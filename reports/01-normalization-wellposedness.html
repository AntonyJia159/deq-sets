Set-DEQ Notes · #1

Path Independence, Exact Unlearning, and the Quiet Role of Normalization in Set-DEQs

Zhibai Jia · 19 June 2026 · working notes, not for circulation beyond reviewers

1. The idea in one paragraph

A Deep Equilibrium Network (DEQ) replaces a deep stack of layers with a single layer iterated to a fixed point Z* = f(Z*, X). I am applying this to set-structured data: the input X is a set of feature vectors (here, points in a feature space), and the model iterates a permutation-equivariant update until the set's representation settles into an equilibrium. The appeal is a cluster of properties that a fixed point gives almost by definition: path independence (any initialization converges to the same solution), exact unlearning / erasability (remove a point, re-solve, and the result is identical to never having seen it — because the fixed point is a pure function of the current set), and test-time scaling (more solver iterations → more compute → handle larger inputs). This note reports the very first probe: do these properties come for free, or do they need design?

Headline finding. The properties are not free. But the decisive ingredient is ordinary normalization (LayerNorm), not the spectral / contractivity machinery the DEQ-theory literature emphasizes. Normalization buys a well-posed, non-diverging fixed point; the choice of aggregator (attention vs. mean-pool) separately buys task accuracy. These two axes turn out to be cleanly separable.

2. The setup

Task

A deliberately simple, controllable synthetic task: count the clusters. Each sample is a set of N = 24 points in 2-D drawn from k well-separated Gaussian blobs, k ∈ {1,2,3,4}; the label is k. The task genuinely needs iterative, mutual refinement among points (a point's cluster identity depends on the whole configuration), so it exercises the equilibrium dynamics rather than a single feed-forward pass. 2000 training sets, 500 test sets.

Model

The latent Z has one vector per input point, shape (N, d_latent) with d_latent = 32. One update step is iterated to a fixed point; a permutation-invariant read-out (mean-pool over points → small MLP) produces the class. Three update blocks are compared:

  • DeepSets (raw): z_i ← MLP([ z_i , x_i , mean_j z_j ]). Points interact only through the mean of all latents. No normalization. One hidden layer.
  • DeepSets + spectral norm: identical, but each linear layer is spectrally normalized (the textbook contractivity knob).
  • NormDeepSets (control): the same mean-pool aggregator, but wrapped in the same structure as a transformer block — input injection, residual connections, two LayerNorms, and a feed-forward layer.
  • Attention: a single multi-head self-attention block (4 heads) with the same residual + LayerNorm + feed-forward wrapper. Points interact through full pairwise attention.

The control pair is the point of the design: NormDeepSets and Attention are identical except for the mixing operation, and DeepSets (raw) vs NormDeepSets are identical except for normalization. So each comparison changes exactly one thing.

What the metrics mean

  • Test accuracy — cluster-count classification accuracy on held-out sets.
  • fp_gap — solve from several random initializations; the largest relative distance between the resulting fixed points. Latent-space path dependence. Small = unique.
  • pred_agreement — fraction of random initializations that yield the same predicted class. Decision-level path dependence. 1.0 = the answer never depends on init.
  • unlearn_gap — remove one point, then compare a warm-started re-solve (from the old fixed point minus that row) against a cold from-scratch solve. Near-zero = exact unlearning; a large value means the warm start clung to a stale, history-dependent state (the leakage failure mode). This is the illusion-detector.
  • warm / cold iters — solver iterations to converge after a removal. warm < cold is the efficiency bonus (and a hint that warm-starting is sound).

Probes use a tight solver (up to 200 iterations, tolerance 1e-6) so that reported fixed points are genuinely converged, not artifacts of an early stop.

Reading the table

Normalization → well-posedness. The raw DeepSets update has no fixed point in practice: the iteration never converges (it sits at the 200-iteration cap), the latent diverges to enormous norms, and the "unlearning gap" of 10⁵–10⁷ is simply divergence noise. Adding normalization while keeping the same mean-pool aggregator (NormDeepSets) flips this completely: the solver converges and unlearning becomes near-exact (gap 0.025). The aggregator was never the cause of the divergence.

Spectral norm alone does not fix it — and is in fact worse here. Bounding each weight's spectral radius to 1 is insufficient because the update consumes both z and the pooled z (Lipschitz constant ~2×), and the composition with ReLU compounds it. So the textbook contractivity knob, applied naively, does not deliver a well-posed fixed point.

Attention → accuracy. Holding normalization fixed and changing only the mixing (NormDeepSets → Attention), accuracy rises 0.75 → 0.83. Pairwise attention models the interactive cluster dynamics that a single mean-pool smears away.

The clean dissociation. Two axes, separable:
  • Normalization controls whether an equilibrium exists and is well-behaved (no divergence; approximately path-independent; unlearnable).
  • The aggregator controls task performance.

An honest caveat, and a new hypothesis

Normalization guarantees existence and boundedness — intuitively it confines the dynamics to a bounded region, so a fixed point must exist — but it does not guarantee uniqueness. The small residual non-uniqueness we still see (fp_gap = 0.033, pred_agreement = 0.98 for attention) is real, not noise. That residual is exactly the regime where an adversary could try to recover a removed point, and where the next phase of this work (membership-inference evaluation and noise masking) will focus.

A bonus observation, offered as a hypothesis rather than a claim: NormDeepSets is actually cleaner on the equilibrium properties (perfect prediction agreement, slightly smaller unlearning gap) despite lower accuracy. This hints at an expressiveness ↔ uniqueness tradeoff — attention's richer fixed points may be marginally less unique, hence marginally more leaky. Worth testing under stress.

4. Where this sits in the literature

LayerNorm is known to be load-bearing in DEQs — the original Deep Equilibrium Models paper notes it "stabilizes" training, and later work reports that DEQs "diverge without layer normalization." But it has been treated as a practical brittleness to be patched, while the theory of well-posed DEQs goes through spectral, monotone-operator, and Jacobian-regularization constraints. The framing that appears under-explored — and that these experiments motivate — is the explicit separation of existence/boundedness (which normalization provides) from uniqueness (which requires contractivity), specifically in the permutation-invariant set setting, and tied to the unlearning property.

All cited claims are to be verified against the primary papers before any formal write-up.

5. Next steps

  • Cardinality shift (train at N=24, probe at N=100): the core "train small, infer large" claim, and the most likely place these properties first crack. Boundedness should survive; uniqueness may not.
  • Harden the instrument: explicit divergence detection and a "% converged" statistic; more initializations and an entropy-based path-dependence measure.
  • Adversarial phase: membership-inference attack as the unlearning-quality metric; differential-privacy noise masking of the residual leak.
